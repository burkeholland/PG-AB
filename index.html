PhoneGap vs. Cordova vs. AppBuilder

What's the difference?

AppBuilder

Telerik AppBuilder is a cross-platform set of tools to build apps for iOS, Android and Windows Phone using an instantly available, hyper productive PhoneGap/Cordova development environment.

Cordova

Apache Cordova is a set of device APIs that allow a mobile app developer to access native device function such as the camera or accelerometer from JavaScript. Combined with a UI framework, this allows a smartphone app to be developed with just HTML, CSS, and JavaScript.

PhoneGap

PhoneGap is an open source solution for building cross-platform mobile apps with standards-based Web technologies like HTML, JavaScript, CSS.

Based on Cordova

Phonegap is a distribution of Apache Cordova. Cordova powers PhoneGap, just like WebKit powers Chrome and Safari. You can build PhoneGap apps using the Command Line Interface (CLI). AppBuilder is a platform that supports Cordova APIs and many different ways of building and distributing apps. With AppBuilder, you can use Cordova core plugins like the accelerometer, camera, compass or the file system to access cross-platform native functionality in your AppBuilder-built app. You also can use community-built Cordova custom plugins for add-on functionality such as bar code scanners and iBeacon readers from the curated Telerik Plugin Marketplace.

Build

What's the difference between PhoneGap Build and AppBuilder?

PhoneGap Build

PhoneGap Build is a service associated with PhoneGap. Using PhoneGap Build, the developer uploads a project's web assets either in ZIP format or a single page, and the service compiles and packages an app. The developer then receives the download URLs for all platforms to test the app on a device. Alternately, the developer can configure the service to point to an opensource git repository and perform builds from that endpoint. According to the type of app being built (opensource or private), the number of apps and the plugins that can be integrated may be limited.

AppBuilder

AppBuilder is an end-to-end solution to deliver mobile apps from concept to App store. You can develop apps using the CLI, the in-browser client, Sublime Text plugins, Visual Studio plugins or the Windows client. AppBuilder includes a large array of tools to aid in the building and distribution of mobile apps, including an in-browser JavaScript-based authoring tool, the ability to simultaneously build a mobile website alongside your app, backend services for data storage, and add-on modules for Analytics, App Feedback, and Prototyping. It is designed to be a one-stop shop for mobile app building, testing, and delivery to production.

Sync

How do you refresh your app to test it on a device using PhoneGap vs. AppBuilder?

PhoneGap Build Hydration

PhoneGap Build is excited to offer Hydration to its users. Using Hydration, updates are pushed directly to the application installed on a device. PhoneGap accomplishes this by compiling a native binary that acts as a container for your mobile application. Once a developer uploads a new build the end user of the application will be notified when restarting the application. If the end user decides to run the new code base, the Hydrated app will automatically fetch and run the latest code base.

AppBuilder LiveSync

LiveSync enables developers to see code changes they make in real-time in both the Icenium Device Simulator and any connected mobile devices. There is no need to rebuild and redeploy the project; nor is there a need to reload the project on the devices. When you deploy your application on connected physical or virtual devices, you can refresh your changes automatically by saving your changes.When you deploy your application in the AppBuilder companion app on remote devices, you can refresh your changes by tapping and holding three fingers.Note, for changes to plugins, configurations, or base assets like icons, you do need to rebuild and refresh the app on your device as live sync won't pick up those changes.

Test

How can you test a mobile app using PhoneGap vs. AppBuilder?

PhoneGap Emulation

There are several ways to test an app's behavior. By typing 'phonegap emulate ios' you can see how your app looks on an iOS simulator, and similarly for Android emulators. PhoneGap Build offers a quick way to download builds for testing on a device, but you do need to include provisioning profiles for your iOS testers.

AppBuilder

AppBuilder offers out-of-the-box a large selection of simulation options, including many device sizes and types and Windows Phone. You can both simulate an app's behavior using AppBuilder's built-in simulators or emulate the app using native simulators associated with Android, Windows Phone, and Xcode. In addition, when you're ready to test on device, AppBuilder offers the Companion App, a way for you to test your app on a device without provisioning, which eases iOS development in particular. Finally, you can build and deploy for device from either the CLI or AppBuilder's in-browser toolset.

Debug

How can you debug a mobile app using PhoneGap vs. AppBuilder?

PhoneGap Debugging with Ripple

PhoneGap offers an emulation service at emulate.phonegap.com where you can enter a url for the service to emulate. This emulation requires the Chrome browser and the Ripple emulation extension to be installed. Alternately, simply using Chrome's Dev Tools with the Device Emulation extension installed helps considerably to test your app.

AppBuilder Debugging Options

Appbuilder includes several options for debugging yor app. First, it includes the ability to check javascript performance by inspecting the code in a Chrome browser on your local machine similar to PhoneGap's. Second, AppBuilder offers a suite of sophisticated testing tools. The new Test Portal allows developers to write JavaScript test scripts and run them on devices in the cloud, greatly expanding the number of virtual devices on which to test apps.
